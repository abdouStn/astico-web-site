Les répertoires:
	/app
		contient tout ce qui concerne votre site Internet… sauf son code source (cache, logs ...)
	/src
		code source. organisé en bundles (des briques de notre application,)
	/vendor
		 les bibliothèques externes à notre application. (Doctrine, Twig, SwiftMailer ...)	
	/web
		les fichiers destinés à vos visiteurs : images, fichiers CSS et JavaScript. Egalement  le contrôleur frontal (point d'entrée: 		app.php et app_dev.php).
		 le seul répertoire qui devrait être accessible à vos visiteurs.
######################################################################
La console
	se placer /var/www/hmtl/symfony , ouvrir le terminal. la commande pour la console de symfony est "php app/console"
	Depuis cette console, on pourra par exemple créer une base de données, vider le cache, ajouter ou modifier des utilisateurs (sans 		passer par phpMyAdmin !) ...
	
	chaque partie de votre site est un bundle. pour en creer un faire "php app/console generate:bundle"
		je rentre "SNS/PlateformBundle" pour creér le bundle de la page d'accueil du site (Bundle obligatoire en suffixe).
		si c'est pas fait automatiquement, dire au routeur principal de chargeur les routes de notre nouveau bundle.
		(dans app/config/	routing.yml).
		Je peux tester mon hello world ainsi:
		http://astico.local/app_dev.php/hello/world       (http://localhost/Symfony/web/app_dev.php/hello/World  en générale)
		
		vider le cache:
		php app/console cache:clear --env=prod
		php app/console cache:clear
